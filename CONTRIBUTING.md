# ⚔️ Rules of Engagement: Contributing to Scoreboards

First of all, thank you for enlisting. By contributing to this repository, you are helping build the most accurate open-source camerounians football database in the world.

To maintain our **Apps** standards, please follow these protocols.

## 📡 Reporting Intelligence Errors (Bugs)

If the system is not behaving as expected:

1. **Check Existing Logs**: Ensure the issue hasn't already been reported in the GitHub Issues tab.
2. **Open a Protocol (Issue)**: Use a clear title (e.g., `[BUG] Overflow on MatchCard at 320px width`).
3. **Provide Specs**: Include your device model, OS version, and steps to reproduce.

## 🛠️ Code Contributions

We welcome tactical improvements. To submit a Pull Request:

1. **Fork the Repository**: Create your own tactical branch.
2. **Adhere to the Aesthetic**:
   - Use our high-contrast color palette.
   - Follow the **Black/Bold Uppercase** typography rules.
   - Ensure all components are `rounded-xl` or higher.
3. **Clean Code Protocol**:
   - Ensure all new widgets are documented.
   - Run `flutter analyze` before committing.
4. **Submit for Review**: Tag your PR with `[FEATURE]` or `[FIX]`.

## 🛡️ Becoming a Sentinel (Data Management)

This project relies on manual data entry for many leagues. If you want to help manage match data, lineups, or historical editions:

1. Contact us via the **Support Page** in the web app.
2. Request "Sentinel Status" for your preferred league.
3. Once verified, you will be given administrative access to the Management Console.
